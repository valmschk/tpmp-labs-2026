## Контрольные вопросы по Лабораторной работе №2

### 1. Приведите примеры облачных IDE
AWS Cloud9, CodeTasty, Gitpod, Repl.it, JSFiddle, Arduino Web Editor, Codeanywhere.

### 2. Опишите процесс подключения репозитория в repl.it
1. Авторизация на сайте через аккаунт GitHub.
2. Нажатие кнопки **«Create Repl»** и выбор вкладки **«Import from GitHub»**.
3. Вставка ссылки на нужный репозиторий.
4. Выбор языка программирования и конфигурация команды запуска.

### 3. Что такое «сборка»?
**Сборка** — это процесс преобразования исходного кода и ресурсов в готовый программный продукт (исполняемый файл, библиотеку или пакет).

### 4. Какие утилиты могут использоваться для сборки?
Компиляторы (**gcc**, **clang**), системы автоматизации (**make**, **CMake**, **Gradle**, **Ninja**, **Ant**).

### 5. Для чего служит Makefile?
**Makefile** — это конфигурационный файл для утилиты `make`, содержащий инструкции (правила) о том, как именно нужно компилировать и связывать файлы проекта. Он автоматизирует процесс сборки, экономя время за счет пересборки только измененных файлов.

### 6. Что такое цель в Makefile? Приведите пример.
**Цель (target)** — это результат, который должен быть получен (имя файла или название действия).
* *Пример:* `main.o` (цель-файл) или `all` (абстрактная цель).

### 7. Что такое связка в Makefile? Приведите пример.
**Связка** — это законченный блок, состоящий из цели, её зависимостей и команд (правил) для выполнения.
* *Пример:*
    ```makefile
    main: main.o
        gcc main.o -o main
    ```

### 8. Что такое зависимость в Makefile? Приведите пример.
**Зависимости (prerequisites)** — это файлы или другие цели, которые должны существовать до того, как начнется сборка текущей цели.
* *Пример:* в строке `main: main.c process.h` файлы `main.c` и `process.h` являются зависимостями для цели `main`.

### 9. Что такое правило в Makefile? Приведите пример.
**Правила (recipes)** — это команды оболочки, которые выполняют сборку. Они обязательно должны начинаться с символа **Tab**.
* *Пример:* `gcc -c main.c -o main.o`

### 10. Что такое макроопределение в Makefile? Приведите пример.
**Макроопределения (переменные)** — это идентификаторы для хранения строк текста (имена компиляторов, флаги, списки файлов).
* *Пример:* `CC = gcc`. Обращение к нему в коде: `$(CC)`.

### 11. Какая связка используется для очистки проекта?
Связка с целью **clean**. Она удаляет временные объектные файлы и исполняемые файлы.
* *Пример:*
    ```makefile
    clean:
        rm -rf *.o main
    ```

### 12. В чем заключается модель КИС для проектов на языке C?
Модель **КИС (Клиент-Интерфейс-Сервер)** — это архитектурный подход к разделению кода:
* **Клиент:** Главный файл (`main.c`), управляющий логикой.
* **Интерфейс:** Заголовочные файлы (`.h`), описывающие прототипы функций.
* **Сервер:** Файлы реализации (`.c`), содержащие логику функций.

### 13. Что такое стандарты кодирования и почему рекомендуется им следовать?
Это набор правил оформления кода (отступы, именование). Им следуют для повышения читаемости кода, уменьшения ошибок и облегчения совместной работы.

### 14. Какой должна быть структура проекта согласно модели КИС и правилам сборки?
* `/src` — исходные коды (`.c`).
* `/include` — заголовочные файлы (`.h`).
* `/docs` — документация и тесты.
* `Makefile` и `README.md` — в корне проекта.

### 15. Что такое GitHub Actions и для каких задач применяется данный функционал?
Это система непрерывной интеграции и доставки (CI/CD). Применяется для автоматического тестирования, сборки и деплоя приложений прямо в репозитории.

### 16. Что такое workflow в GitHub Actions? Приведите пример.
**Workflow** — это автоматизированный процесс, описанный в `.yml` файле.
* *Пример:* Сценарий `c-build.yml` для автоматической компиляции проекта.

### 17. Что такое event в GitHub Actions? Приведите пример.
**Event** — это событие, вызывающее запуск процесса.
* *Пример:* `push` (отправка кода), `pull_request`.

### 18. Что такое job в GitHub Actions? Приведите пример.
**Job** — это группа шагов, выполняемая на одном виртуальном сервере.
* *Пример:* Задание `build` или `test_suite`.

### 19. Что такое step в GitHub Actions? Приведите пример.
**Step** — это конкретная задача внутри Job (команда или действие).
* *Пример:* `run: make` или `uses: actions/checkout@v3`.

### 20. Что такое action в GitHub Actions? Приведите пример.
**Action** — это готовый многократно используемый блок кода для типовых задач.
* *Пример:* `actions/checkout@v3` (клонирование кода в виртуальную среду).
